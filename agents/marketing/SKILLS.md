# LeadFlow Marketing Agent - SKILLS.md

## Core Skills

### 1. research_target_agents
**Purpose:** Find real estate agents who match pilot profile
**Input:** Location, agent type, channel
**Output:** List of 10-20 qualified agents with contact info
**Tools:**
- Facebook group search
- LinkedIn Sales Navigator
- Real estate agent directories
**Acceptance:**
- [ ] Agent is independent or team lead (not large brokerage)
- [ ] Agent has online presence (active on social)
- [ ] Contact method identified (DM, email, etc.)
- [ ] Personalized icebreaker noted

### 2. craft_outreach_message
**Purpose:** Write personalized, pain-first outreach
**Input:** Agent profile, channel, pain point
**Output:** Message ready to send
**Rules:**
- Lead with pain, not features
- Keep under 100 words
- Include clear CTA ("Wanna see the demo?")
- Match tone to channel (casual for FB, professional for LI)
**Acceptance:**
- [ ] Message mentions specific pain point
- [ ] No feature dumping
- [ ] Clear next step
- [ ] Tone appropriate for channel

### 3. send_outreach
**Purpose:** Send message and track in spreadsheet
**Input:** Message, contact method, agent info
**Output:** Message sent, logged
**Tools:**
- Facebook Messenger
- LinkedIn DM
- Email
- Tracking spreadsheet (Google Sheets)
**Acceptance:**
- [ ] Message sent
- [ ] Logged in tracking sheet
- [ ] Follow-up reminder set (3 days)

### 4. handle_response
**Purpose:** Reply to agent inquiries, move toward demo
**Input:** Agent message
**Output:** Reply sent, demo scheduled or objection handled
**Common Responses:**
- "Tell me more" → Send demo link
- "How much?" → "Free for 30 days during pilot"
- "I'm busy" → "Demo takes 5 min, when works?"
- "Not interested" → Ask why, note for future
**Acceptance:**
- [ ] Response sent within 2 hours
- [ ] Conversation logged
- [ ] Demo scheduled OR reason noted

### 5. schedule_demo
**Purpose:** Get agent on a call to see the product
**Input:** Agent interest, availability
**Output:** Cal.com booking confirmed
**Tools:**
- Cal.com booking link
- Loom video (async option)
**Acceptance:**
- [ ] Demo scheduled
- [ ] Calendar invite sent
- [ ] Prep notes added (agent's pain points)

### 6. conduct_demo
**Purpose:** Show working product, answer questions
**Input:** Agent profile, pain points
**Output:** Demo completed, feedback gathered
**Demo Flow:**
1. "What pain are you feeling with leads?" (2 min)
2. Show dashboard, live leads (3 min)
3. Show SMS conversation (2 min)
4. "How would this change your business?" (3 min)
5. Pilot sign-up (if interested)
**Acceptance:**
- [ ] Agent saw working product
- [ ] Pain points validated
- [ ] Pilot offer made
- [ ] Feedback noted

### 7. close_pilot
**Purpose:** Get agent to sign pilot agreement
**Input:** Demo completed, agent interested
**Output:** Signed pilot agreement
**Process:**
1. Send pilot agreement (Google Doc)
2. Explain: 30 days free, feedback required, can cancel anytime
3. Get verbal/email confirmation
4. Add to pilot tracking sheet
**Acceptance:**
- [ ] Pilot agreement signed
- [ ] FUB connection instructions sent
- [ ] Onboarding scheduled
- [ ] Added to pilot tracking

### 8. ask_for_referrals
**Purpose:** Get introductions to other agents
**Input:** Happy pilot agent
**Output:** 2-3 referral names
**Script:**
> "Who else do you know who's drowning in leads and losing money to slow response times? I'd love to help them too."
**Incentive:** 1 month free for each referral who signs
**Acceptance:**
- [ ] Referral ask made
- [ ] Names collected
- [ ] Introductions requested
- [ ] Incentive explained

### 9. track_metrics
**Purpose:** Monitor outreach effectiveness
**Input:** Daily activities
**Output:** Updated metrics dashboard
**Metrics to Track:**
- Outreach sent
- Response rate
- Demos scheduled
- Demos completed
- Pilots signed
- Referrals received
**Acceptance:**
- [ ] Daily log updated
- [ ] Metrics calculated
- [ ] Patterns noted (what's working)

### 10. report_blockers
**Purpose:** Alert orchestrator when stuck
**Input:** Problem encountered
**Output:** Blocker documented, escalated if needed
**Blocker Types:**
- No responses (messaging issue)
- Demos but no sign-ups (value unclear)
- Technical questions I can't answer
**Acceptance:**
- [ ] Blocker documented
- [ ] Attempted solutions noted
- [ ] Escalated to orchestrator if needed
