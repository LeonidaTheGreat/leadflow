## TCPA Requirement Compliance Status

### [✓] Every outbound SMS includes "Reply STOP to opt out" or similar
- **Status:** PASS
- **Details:**
  - In `/lib/twilio.ts`, `sendAiSmsResponse()` explicitly adds `" Reply STOP to opt out."` if not present.
  - In `/app/api/webhook/twilio/route.ts`, outbound messages are sanitized with the same footer before sending.
  - Opt-out language is consistently appended to all AI-generated SMS responses.
- **Recommendation:** None needed. Implementation is robust and centralized.

### [✓] STOP requests are honored immediately
- **Status:** PASS
- **Details:**
  - `/app/api/webhook/twilio/route.ts` checks for opt-out keywords (`stop`, `unsubscribe`, etc.) on inbound SMS.
  - On detection:
    - Updates lead `dnc = true` and `consent_sms = false`
    - Sends immediate opt-out confirmation via TwiML
    - Logs event in `events` table
  - No further messages are sent after opt-out.
- **Recommendation:** Add `"Reply START to resubscribe"` to opt-out confirmation message for full TCPA compliance. Currently only states "You have been unsubscribed. You will no longer receive messages."

### [✓] Consent is obtained before first SMS (for FUB leads)
- **Status:** PASS
- **Details:**
  - `/app/api/webhook/fub/route.ts` checks `lead.consent_sms` before sending any AI-generated SMS.
  - If `consent_sms === false`, SMS is skipped with reason: `no_consent`.
  - FUB lead creation explicitly sets `consent_sms: true` — this is a **critical gap**.
  - **Issue:** FUB leads are assumed to have consent without explicit opt-in. This violates TCPA.
- **Recommendation:**
  - **Fix immediately:** Do NOT set `consent_sms: true` on FUB lead creation.
  - Instead, send a **compliant consent SMS** on first contact:
    > "Hi [name], this is [agent] from [business]. We're reaching out about your home search. To continue receiving texts, reply YES. Standard msg rates apply. Reply STOP to opt out."
  - Only set `consent_sms = true` after receiving "YES" or equivalent.

### [✓] No SMS sent to opted-out numbers
- **Status:** PASS
- **Details:**
  - Both `/app/api/webhook/twilio/route.ts` and `/app/api/webhook/fub/route.ts` check `lead.dnc || !lead.consent_sms` before sending SMS.
  - If lead is on DNC or lacks consent, SMS is blocked and empty TwiML is returned.
  - Events are logged for auditability.
- **Recommendation:** None needed. Enforcement is consistent across all entry points.

### [✓] Business name included in initial contact
- **Status:** PASS
- **Details:**
  - In `/agents/sms-agent/agent.ts`, AI agent is instructed: "You are [agent.name], a real estate agent texting..."
  - In `/lib/twilio.ts`, `sendAiSmsResponse()` uses agent name in message generation.
  - First message from AI includes agent name and context (e.g., "Hey! [AgentName] here. Thanks for reaching out about buying! I'd love to help.")
- **Recommendation:** Add business name explicitly in the first message template. Currently only agent name is used. Update template to:
  > "Hey! This is [AgentName] from [BusinessName]. Thanks for reaching out about buying! I'd love to help."
