## Executive Summary

Fixed column name mismatches in API routes following the migration from Supabase SDK to PostgREST HTTP client. The most critical mismatch was in the `agents/profile/route.ts` file which was referencing a non-existent column name `photo_url` when the actual database column in `agent_profiles` table is `profile_image`.

---

## Root Cause Analysis

During the Supabase SDK → PostgREST migration, several column references in API routes became misaligned with the actual database schema. The Supabase SDK had abstraction layers that may have masked these issues during development, but the PostgREST HTTP client exposes direct column mismatches.

**Key Finding:** The `agent_profiles` table schema defines:
- `profile_image VARCHAR(500)` — correct column name in DB
- Code was querying `.select('bio, photo_url')` — **incorrect** column name
- Code was upserting `photo_url: ...` — **incorrect** column name
