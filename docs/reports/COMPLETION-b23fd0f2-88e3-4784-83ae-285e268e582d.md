## Implementation Details

### 1. Core API Endpoint

**File:** `product/lead-response/dashboard/app/api/cron/follow-up/route.ts`

The cron handler processes active follow-up sequences hourly and:
- Queries leads with active sequences due for sending
- Filters by DNC status, SMS consent, and frequency caps
- Generates contextual AI responses using lead/agent context
- Adds TCPA-compliant opt-out footer
- Sends SMS via Twilio with status tracking
- Updates sequence state (step, sent count, status)
- Supports dry-run mode for testing

**Key Features:**
- Respects quiet hours (9 PM - 9 AM) to avoid night-time SMS
- Enforces frequency capping (max 3 messages per lead per 24h)
- Auto-pauses sequences on lead inbound response
- Trims long messages to fit SMS 160-character limit with footer
- Handles missing leads/agents gracefully with skip + warning logs
- Returns detailed results (sent count, skip count, failed count)

## Sequence Types & Triggers

### no_response (24h)
- **Trigger:** Lead hasn't replied within 24 hours of outbound message
- **Use case:** Re-engage cold leads
- **Timing:** Initial contact → wait 24h → send follow-up

### post_viewing (4h)
- **Trigger:** Agent confirmed property viewing completion
- **Use case:** Gather feedback and schedule next steps
- **Timing:** Viewing completed → wait 4h → send follow-up

### no_show (30m)
- **Trigger:** Lead missed scheduled appointment
- **Use case:** Reschedule or address issue
- **Timing:** Appointment time passed → wait 30m → send follow-up

### nurture (7d)
- **Trigger:** Lead has no activity for 7 days
- **Use case:** Keep leads engaged during consideration phase
- **Timing:** Last activity → wait 7d → send follow-up

## Integration Points

### Leads Table
- Foreign key: `lead_id` references `leads(id)`
- Used for DNC, consent, and agent context
- Cascade delete removes sequences if lead deleted

### Messages Table
- Each sent follow-up creates a message record
- Direction: `outbound`, Channel: `sms`
- AI confidence and Twilio SID captured for tracking

### Agents Table
- Joined for agent context (name, email, market, settings)
- Used in AI prompt for personalization
- Settings used to configure sequence behavior per agent

### Twilio Integration
- SMS sent via `sendSms()` with status callback
- Twilio SID captured for delivery tracking
- Status callback URL registered in Twilio

### AI Integration
- Uses `generateAiSmsResponse()` for contextual messaging
- Passes lead + agent context to AI for personalization
- AI confidence score captured with message
