## Summary

Unified authentication cookie handling across all protected API routes. Previously, routes inconsistently checked only one of two auth mechanisms:
- `auth-token` cookie (JWT from trial/pilot signup)
- `leadflow_session` cookie (session DB token from login)

All routes now use the `getAuthUserId()` helper from `lib/auth.ts` which checks both auth sources.

## Changes

### New/Updated Library
- **`product/lead-response/dashboard/lib/auth.ts`** — Unified auth helper `getAuthUserId(request)`:
  1. Checks `auth-token` JWT cookie first (trial/pilot signup flow)
  2. Falls back to `leadflow_session` session token (login flow, looks up `sessions` table in Supabase)
  3. Returns `null` if neither valid → route returns 401
