## Root Cause Analysis

The QC agent was failing with `zombie_timeout` repeatedly because the **`npm run build` step was failing with a TypeScript error**, causing the automated gates check to reject immediately:

```
./app/api/nps/dismiss/route.ts:35:35
Type error: Property 'user' does not exist on type 'never'.
```

**Why this error occurred:**

The NPS API routes (`dismiss`, `prompt-status`, `submit`) were using the Supabase auth pattern:
```ts
const supabase = createClient(supabaseUrl, supabaseAnonKey)
const { data: { session }, error: sessionError } = await supabase.auth.getSession()
if (sessionError || !session?.user) { ... }
```

But `lib/db.ts` provides a **custom PostgREST client** (not the real Supabase SDK). Its `auth.getSession()` mock always returns:
```ts
{ data: { session: null }, error: null }
```

TypeScript infers `session` as type `null`. Therefore `session?.user` resolves to type `never`, and accessing `.user` on `never` is a compile-time error.

---

## Fix Applied

Replaced the broken Supabase auth pattern in all 3 NPS routes with the correct project auth pattern:

1. Read `leadflow_session` cookie from the request
2. Validate via `validateSession()` from `@/lib/session`
3. Use `session.userId` directly
