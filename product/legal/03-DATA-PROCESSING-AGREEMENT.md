# Data Processing Agreement (DPA)
## AI Lead Response System for Real Estate Agents

**Effective Date:** February 2026  
**Last Updated:** February 14, 2026

This Data Processing Agreement supplements the Terms of Service and Privacy Policy. It applies to processing of personal data under GDPR, CCPA, and similar regulations.

---

## 1. Purpose and Scope

### 1.1 Roles
- **Data Controller:** The User (real estate agent/agency) determines the purpose and means of personal data processing
- **Data Processor:** Provider (company providing the Service) processes data on behalf of the Controller
- **Sub-Processors:** Third-party service providers process data under the Controller and Processor relationship

### 1.2 Applicable Data
This DPA covers all personal data processed through the Service, including:
- Lead contact information (names, emails, phone numbers)
- Lead property information and communication history
- User account information
- Communication records and metadata
- Any other personal data uploaded by the Controller

### 1.3 Processing Activities
This DPA governs:
- Storage of lead and communication data
- AI processing for response generation
- Integration with CRM systems
- Analytics and usage tracking
- Data retention and deletion

## 2. Data Processor Obligations

### 2.1 Processing Instructions
Provider agrees to:
- Process personal data only on documented instructions from Controller
- Not process data for Provider's own purposes (except as specified in this DPA)
- Document all processing activities
- Implement appropriate technical and organizational measures

### 2.2 Processor Instructions
The Controller instructs Provider to:
- Store lead information on secure servers
- Process lead information through AI models for response generation
- Share lead information with approved sub-processors (see Section 4)
- Retain data according to retention schedules
- Delete data upon request or account termination

### 2.3 Permitted Uses
Provider may use personal data to:
- Provide the AI Lead Response System as contracted
- Maintain and improve service infrastructure
- Comply with legal obligations
- Enforce this DPA and Terms of Service
- Respond to lawful requests from authorities

### 2.4 Prohibited Uses
Provider shall NOT:
- Sell personal data to third parties
- Use personal data for competitive purposes
- Share personal data with unapproved third parties
- Use personal data for marketing Provider's other services
- Combine personal data with data from other sources (except as specified)

## 3. Data Security and Confidentiality

### 3.1 Security Measures
Provider implements appropriate technical and organizational measures, including:

**Infrastructure Security:**
- Secure data centers with physical access controls
- Firewalls and intrusion detection systems
- Network monitoring and threat detection
- Regular security audits and penetration testing

**Data Protection:**
- Encryption in transit (SSL/TLS 1.2+)
- Encryption at rest for sensitive data (AES-256)
- Secure key management and rotation
- Access logging and audit trails

**Access Control:**
- Role-based access control (RBAC)
- Multi-factor authentication for administrative access
- Principle of least privilege
- Employee confidentiality agreements

**Monitoring and Response:**
- 24/7 security monitoring
- Incident response procedures
- Regular backup and disaster recovery testing
- Data breach notification procedures

### 3.2 Employee Confidentiality
Provider ensures all employees and contractors:
- Sign confidentiality agreements
- Receive data protection training
- Understand GDPR and CCPA requirements
- Comply with data handling policies

### 3.3 Subprocessor Security
Provider ensures all sub-processors:
- Implement equivalent security measures
- Sign data processing agreements
- Maintain security compliance certifications
- Cooperate with security audits

### 3.4 Security Certifications
Provider maintains:
- SOC 2 Type II certification (or equivalent)
- Regular security audits
- Industry compliance standards
- Documentation available upon request

## 5. Data Subject Rights

### 5.1 Right to Access
Upon request from a data subject (lead), Controller is responsible for:
- Confirming identity
- Retrieving information from the Service
- Responding within timeframes required by law

Provider will:
- Provide data in machine-readable format upon request
- Cooperate with Controller in fulfilling requests
- Not impose unreasonable delays

### 5.2 Right to Deletion
Upon valid deletion request, Provider will:
- Delete personal data from live systems within 30 days
- Delete from backups within 90 days
- Maintain records only if required by law
- Verify deletion completion

### 5.3 Right to Correction
Upon notification of inaccurate data, Provider will:
- Implement corrections
- Notify affected sub-processors
- Maintain audit trail of corrections

### 5.4 Right to Portability
Provider will:
- Export data in structured, portable format upon request
- Provide data to alternative processors upon request
- Cooperate in data transfer process

## 9. Data Retention and Deletion

### 9.1 Retention Schedule
Data is retained according to the following schedule:

| Data Type | Retention Period | Rationale |
|---|---|---|
| Lead Information | Active account + 90 days after deletion | Service provision and dispute resolution |
| Communication History | 24 months | Compliance and audit trail |
| Account Information | Account active + 30 days after deletion | Service provision |
| Server Logs | 90 days | Security and troubleshooting |
| Analytics Data | 36 months (aggregated/anonymized) | Performance tracking |
| Backups | 30 days after deletion | Disaster recovery |

### 9.2 Deletion Procedures
- Upon account termination, Controller may request immediate deletion
- Provider will delete data from live systems within 30 days
- Data in backups will be deleted within 90 days
- Deletion confirmation provided upon request

### 9.3 Legal Holds
Provider may retain data if:
- Required by law (legal process, regulatory requirement)
- Necessary for legal defense
- Required for financial or tax records
- Otherwise permitted by law

## 10. Data Breach Notification

### 10.1 Breach Notification
Provider will notify Controller of any personal data breach:
- Without unreasonable delay

- By email to primary contact
- Including affected data and categories

### 10.2 Breach Information
Notification will include:
- Description of the breach
- Likely consequences for data subjects
- Categories and volume of data affected
- Measures taken to address the breach
- Recommended actions for Controller

### 10.3 Controller Responsibilities
- Controller is responsible for notifying data subjects if required by law
- Controller is responsible for notifying authorities
- Provider will cooperate with investigations
- Provider will document breach details for Controller

### 10.4 Breach Investigations
- Provider will investigate all suspected breaches
- Controller will cooperate in investigations
- Root cause analysis will be provided
